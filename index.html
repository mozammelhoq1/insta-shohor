How does javascript works?

JavaScript is a client-side scripting language. Client Side Scripting Language is the ability to show the client everything dynamically. JavaScript is used to show clients everything on the web, but some modern browsers require some engines to support JavaScript. Every Web browser comes with a JavaScript engine that provides a JavaScript runtime environment. For example, Google Chrome uses the V8 JavaScript engine, developed by Lars Bak. V8 is an open-source JavaScript engine developed for Google Chrome and Chromium web browsers by The Chromium Project. The browser’s built-in interpreter searches for script tag or .js file linked with an HTML file while loading a web page, and then interpretation and execution starts. Javascript uses something called Eventloop for Asynchronous calls. The setTimeout is pushed to EventLoop since it is a callback. and the main thread continues to execute. Once the main completes, Then the EventLoop pushes the data to the main stack.

Difference between local storage and Sessional storage

Local storage : This read-only interface property provides access to the Document’s local storage object, the stored data is stored across browser sessions. Similar to sessionStorage, except that localStorage data gets cleared when the page session ends – that is when the page is closed. It is cleared when the last “private” tab of a browser is closed .
Sessional storage : Sessional Storage objects can be accessed using the sessionStorage read-only property.Sessional storage is temporary, the data of the session storage will be stored for as long as the user is in the browser, then the session storage will be closed when the browser closes and the user leaves the browser.
The difference between sessional Storage and localStorage is that localStorage data does not expire, whereas sessionStorage data is cleared when the page session ends.
